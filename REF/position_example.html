📌 예제 1: 기본 상태 vs Relative

Static
Relative
Static
🔍 관찰해보세요:
• 빨간색 박스들(Static)은 원래 자리에 그대로
• 파란색 박스(Relative)는 원래 자리에서 살짝 이동했지만, 원래 공간은 그대로 유지
• 다른 박스들이 파란색 박스의 원래 자리를 침범하지 않음

📌 예제 2: Z-index 마법!

z-index

🎨 z-index의 비밀:
• position: static에서는 z-index가 작동하지 않음
• position: relative에서는 z-index가 작동함!
• 초록색 박스가 다른 박스들 위에 떠있는 것을 확인하세요

📌 예제 3: 부모 역할하는 Relative

부모 (relative)
자식
(absolute)
👨‍👩‍👧‍👦 부모-자식 관계:
• 부모가 position: relative일 때
• 자식의 position: absolute는 부모를 기준으로 위치 설정
• 보라색 박스가 오렌지 박스의 오른쪽 아래 모서리에 위치함

📌 KB 프로젝트에서 왜 사용했을까?

🏢 KyongBo Electric 웹사이트에서의 역할:

.main-content { position: relative; z-index: 1; }

1️⃣ Z-index 활성화: 다른 요소들과의 레이어 순서 관리
2️⃣ 기준점 제공: 내부 absolute 요소들의 부모 역할
3️⃣ 안전장치: 헤더나 다른 고정 요소들과의 충돌 방지

"원래 자리는 지키면서, 필요할 때 앞뒤로 나설 수 있는 능력을 준 것!"

🎓 핵심 정리

Position: Relative = "착한 이동"

✅ 원래 자리는 그대로 유지 (다른 요소들이 침범 안함)
✅ top, right, bottom, left로 미세 조정 가능
✅ z-index 사용 가능 (레이어 순서 조정)
✅ 자식 absolute 요소들의 기준점 역할
✅ 전체 레이아웃을 깨뜨리지 않음

💡 기억하세요: "내 자리는 지키면서, 조금씩 움직일 수 있는 마법!"
